# Выступление для защиты дипломного проекта
## Тема: «Разработка мобильного приложения для прослушивания и оценки музыки (LumiSound)»

---

## Вступительная часть

Добрый день, уважаемые члены комиссии! Меня зовут Гилевич Мирослав. Представляю вашему вниманию дипломный проект на тему «Разработка мобильного приложения для прослушивания и оценки музыки LumiSound».

Актуальность темы обусловлена тем, что современные музыкальные стриминговые сервисы — Spotify, Apple Music, YouTube Music — предоставляют пользователям лишь базовые инструменты оценки: отметки «нравится» или числовой рейтинг. При этом слушатели, особенно увлечённые музыкой, хотят выражать развёрнутое мнение о треке по конкретным критериям. Разработанное приложение LumiSound закрывает этот пробел, совмещая полноценный музыкальный плеер с социальной платформой для глубокого рецензирования.

Целью проекта является разработка мобильного Android-приложения, обеспечивающего прослушивание музыки, её многокритериальную оценку, написание рецензий и комментариев, создание плейлистов, социальное взаимодействие пользователей и административное управление через веб-панель.

Для достижения цели были решены следующие задачи:
- реализовать систему регистрации, авторизации и настройки профиля пользователя;
- интегрировать открытый музыкальный API платформы Audius для получения треков и исполнителей;
- разработать полноэкранный плеер с управлением воспроизведением, waveform-прогресс-баром и плавающими комментариями;
- реализовать TikTok-подобный вертикальный фид для открытия музыки;
- разработать систему оценок по пяти критериям с написанием рецензий и голосованием за авторитет;
- реализовать создание плейлистов и функцию синтез-сессий для совместного составления плейлиста;
- разработать систему жалоб, банов и апелляций;
- создать веб-панель администратора для управления контентом и пользователями;
- добавить настройки: эквалайзер, таймер сна, темы оформления и выбор иконки приложения.

Практическая значимость работы состоит в том, что приложение позволяет меломанам формировать и публично выражать развёрнутое мнение о музыке, находить единомышленников через систему рецензий, открывать новые треки через персонализированный фид, а также самостоятельно добавлять авторский контент через административную панель.

Приложение разработано на языке Kotlin с использованием Jetpack Compose для построения пользовательского интерфейса. Архитектурный паттерн — MVVM. Для воспроизведения аудио используется ExoPlayer. Музыкальный контент поставляется через Audius API, пользовательские данные хранятся в облачной базе данных Supabase на основе PostgreSQL. Внедрение зависимостей реализовано через Hilt.

- Экран приветствия, регистрация и авторизация
- Настройка профиля при первом входе

- Экран поиска: поиск треков и исполнителей, TikTok-фид

- Экран исполнителя: биография, статистика, треки
- Экран рецензий: оценка по критериям, голосование за авторитет
- Создание плейлиста и синтез-сессии
- Экран профиля: статистика, избранное
- Настройки: эквалайзер, таймер сна, темы
- Веб-панель администратора

## Сценарий демонстрации функций приложения

Начнём демонстрацию с экрана приветствия. При первом запуске LumiSound пользователь видит логотип приложения и две кнопки: «Войти» и «Создать аккаунт». Это точка входа для новых и существующих пользователей.

Нажмём «Создать аккаунт». Откроется экран регистрации, где нужно ввести имя пользователя, адрес электронной почты, пароль и его подтверждение. Реализована проверка сложности пароля в реальном времени: три уровня — слабый, хороший, надёжный. Также доступна ссылка на пользовательское соглашение прямо на этом экране. После успешной регистрации система отправляет письмо для подтверждения email. На следующем шаге — экран настройки профиля, где пользователь загружает аватар, вводит отображаемое имя и описание о себе. После завершения настройки пользователь попадает на главный экран.

Войдём в систему через экран авторизации. Нужно ввести email и пароль. Реализовано восстановление пароля по email. После входа происходит предзагрузка данных — приложение загружает профиль, избранные треки, рекомендации — и пользователь попадает на главный экран.

На главном экране отображаются блоки с рекомендованными плейлистами, недавно прослушанными треками и списком любимых исполнителей. В нижней части экрана постоянно присутствует мини-плеер, если воспроизводится трек. Через нижнюю навигационную панель доступны четыре основных раздела: Главная, Поиск, Оценки и Профиль.

Перейдём на экран поиска. Здесь две вкладки: «Рекомендации» и «Для вас». По умолчанию открывается вертикальный TikTok-подобный фид — пользователь листает треки свайпом вверх, каждый трек занимает весь экран. Аудио автоматически начинает играть при переходе на трек, звук можно включить нажатием на экран. На каждой карточке отображаются кнопки для просмотра статистики, написания комментария и рецензии, добавления трека в плейлист. Вкладка «Для вас» показывает треки на основе любимых артистов и плейлистов пользователя. Также можно воспользоваться строкой поиска — она находит треки и исполнителей как в каталоге Audius, так и среди контента, добавленного администратором.

Нажмём на название трека, чтобы открыть полноэкранный плеер. Перед нами обложка трека в высоком разрешении, под ней — waveform-прогресс-бар, отображающий форму волны аудиодорожки. Можно нажать в любую точку прогресс-бара для перемотки. Кнопки управления: предыдущий трек, пауза/воспроизведение, следующий трек. Переключение между треками работает также горизонтальным свайпом. Закрыть плеер можно свайпом вниз. В левом нижнем углу — плавающие комментарии других пользователей, которые появляются поверх обложки. Нажатие на имя исполнителя открывает его профиль.

Откроем профиль исполнителя. На этом экране отображается фото, имя, локация, статистика: количество треков, подписчиков и плейлистов. Ниже — биография и список популярных треков. Для исполнителей, добавленных через администраторскую панель, также доступен полноценный профиль.

Перейдём к рецензиям. Нажмём на иконку звезды на карточке трека в фиде. Откроется экран написания рецензии. Здесь нужно оценить трек по пяти критериям с помощью ползунков от 1 до 10: «Рифмы и образы», «Структура и ритмика», «Реализация стиля», «Индивидуальность и харизма», «Атмосфера и вайб». После выставления оценок можно написать текстовую рецензию. Средняя оценка по всем пяти критериям отображается рядом с названием трека. На странице всех рецензий трека видны рецензии других пользователей, отсортированные по авторитету. Можно проголосовать стрелкой вверх или вниз — авторитет рецензии изменится немедленно, а данные сохранятся на сервере.

Создадим плейлист. На главном экране нажимаем на плейлист — открывается детальный экран с треками. Плейлист можно сделать публичным, редактировать название и обложку. Также доступна функция «Синтез» — это совместный плейлист с другом. Пользователь создаёт сессию, получает код-приглашение и отправляет его другу. После подключения оба участника добавляют треки со своих любимых исполнителей, и система формирует единый плейлист из их совместных предпочтений.

Откроем экран профиля. Здесь отображается аватар, имя, биография, количество оценок и рецензий. Ниже — горизонтальные списки любимых треков и исполнителей, отсортированные по количеству прослушиваний. Можно отредактировать профиль, нажав на иконку карандаша. Через иконку шестерёнки переходим в настройки.

В настройках доступны следующие разделы. Темы оформления — тёмная, светлая и системная. Иконка приложения — можно выбрать из четырёх вариантов: чёрная, белая, розовая и синяя. Воспроизведение — автовоспроизведение, нормализация громкости, скорость воспроизведения. Эквалайзер — настройка частотных полос и выбор пресетов. Таймер сна — автоматически останавливает музыку через заданное время. Очистка кэша. Раздел «О приложении» — с авторскими правами, политикой конфиденциальности и пользовательским соглашением.

Теперь продемонстрируем веб-панель администратора. Это отдельное веб-приложение, доступное через браузер. Для входа нужен администраторский пароль. На дашборде отображается статистика: количество пользователей, артистов, треков, оценок и комментариев.

На вкладке «Пользователи» администратор видит всех зарегистрированных пользователей, может редактировать их данные, блокировать и удалять. На вкладке «Артисты» — управление собственными артистами, загруженными администратором: можно добавить нового артиста, загрузить аватар и обложку. На вкладке «Треки» — управление треками: добавить новый трек, загрузить аудиофайл и обложку — после этого трек появится в поиске приложения. На вкладке «Жалобы» — все жалобы пользователей на комментарии и рецензии с возможностью удалить контент или заблокировать пользователя. На вкладке «Баны» — список заблокированных пользователей. На вкладке «Апелляции» — апелляции заблокированных пользователей с возможностью разбана.

## Подведение итогов

Подведём итог. В ходе работы над проектом было разработано мобильное Android-приложение LumiSound. Реализованы: регистрация и авторизация с подтверждением email, настройка профиля с загрузкой аватара, интеграция с Audius API для получения музыкального контента, полноэкранный плеер с waveform-прогресс-баром и управлением через уведомления, TikTok-подобный персонализированный фид, система оценок по пяти критериям с рецензиями и голосованием за авторитет, комментарии с плавающим отображением в плеере, плейлисты и синтез-сессии, эквалайзер и таймер сна, система жалоб и банов с апелляциями, выбор иконки и темы оформления, а также веб-панель администратора с управлением пользователями, треками и артистами. Все данные надёжно хранятся в облачной базе данных Supabase с разграничением доступа через Row Level Security.
